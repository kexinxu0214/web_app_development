# 算命系統 PRD (Product Requirements Document)

## 1. 專案概述
- **背景與動機**：現代人生活壓力大，時常對感情、事業、健康等方面感到迷惘。許多人有求神問卜的需求，但未必有時間或方便親自前往實體廟宇。本系統希望提供一個線上虛擬平台，讓使用者隨時隨地都能進行算命、占卜與抽籤，獲得心靈上的指引與慰藉。
- **目標用戶**：
  - 對未來感到迷惘，需要尋求方向的現代上班族或學生。
  - 對紫微斗數、塔羅牌、宮廟抽籤等命理玄學感興趣的使用者。
  - 習慣在手機或電腦上進行線上互動、體驗線上祈福的族群。
- **核心價值主張**：提供便捷、有趣且撫慰人心的線上算命體驗，加上儲存結果與捐獻香油錢的機制，打造包含祈福與互動的完整數位生態圈。

## 2. 功能需求
系統預計提供以下主要功能（包含使用者故事）：

1. **線上算命、占卜與抽籤**
   - 作為 **訪客 / 使用者**，我希望 **能在網站上進行不同種類的占卜（如抽籤、擲筊）**，以便 **獲得針對我目前疑惑的解答**。
2. **籤詩與結果解析**
   - 作為 **使用者**，我希望 **在抽籤或占卜後能看到清楚明確的白話文解析**，以便 **我能真正理解占卜結果想傳達的涵義**。
3. **儲存算命結果**
   - 作為 **註冊使用者**，我希望 **系統能自動儲存我過去的抽籤與算命紀錄**，以便 **我日後可以隨時回顧當時的問題與獲得的指引**。
4. **捐香油錢**
   - 作為 **使用者**，我希望 **能在系統內進行線上捐獻香油錢（祈福 / 還願）**，以便 **表達我的感謝並祈求接下來的好運**。
5. **分享算命結果與籤詩**
   - 作為 **使用者**，我希望 **能將算出或抽到的好籤分享到社群媒體（如 LINE、IG）**，以便 **與親朋好友分享我的運勢**。

## 3. 非功能需求
- **技術限制**：
  - 後端：Python Flask
  - 前端：HTML、Vanilla CSS、Jinja2 模板（依據需求建議，暫不使用前端重型框架）
  - 資料庫：SQLite（輕量級、易於開發與部署）
- **效能考量**：
  - 確保快速的頁面載入速度，抽籤或擲筊的微動畫（如有）需平順不卡頓。
  - 確保 RWD (響應式網頁設計) 完善，手機與電腦雙平台操作體驗一致。
- **安全考量**：
  - 使用者密碼需進行雜湊（Hash）加密儲存。
  - 防範常見的網路攻擊（如 SQL Injection、XSS）。
  - （未來若啟用）真實金流串接必須走 HTTPS 安全傳輸，且不自行儲存信用卡敏資。

## 4. MVP 範圍
- **Must Have (必須有)**：
  - 基礎的抽籤 / 占卜互動流程。
  - 籤詩庫與對應的解析顯示。
  - 基礎的使用者系統（註冊、登入、登出）。
  - 將歷史抽籤結果儲存至資料庫的紀錄頁面。
- **Should Have (應該有)**：
  - 香油錢捐獻頁面（MVP 階段可先做虛擬投幣器或感謝動畫，尚未必要直接串接真金流）。
  - 美觀且具備玄學 / 宮廟風格的 UI 設計。
- **Nice to Have (加分項)**：
  - 真正串接第三方支付的香油錢系統。
  - 一鍵生成圖片以便儲存至手機或轉發到社群。
  - 每日運勢推播或每日一籤。
